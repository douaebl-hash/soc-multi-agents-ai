Points clés de l'architecture :

Sans dépendance bloquante : le mode par défaut (use_llm=False) ne repose que sur la bibliothèque standard. Le Small LLM (Ollama, ex. llama3.2:1b) est optionnel, branché via LangChain, et n'intervient qu'en complément des règles sur les messages ambigus — s'il est indisponible, l'agent continue sans planter.
Mémoire partagée fichier-JSON (shared_memory.py) : pattern pub/sub simple par "canaux" (pending_analysis, et bientôt incidents, actions_a_executer pour les agents suivants), remplaçable plus tard par Redis ou une vraie file de messages sans changer l'interface des agents.
Sources simulées réalistes : syslog.log (RFC3164), network_traffic.json (proche Suricata eve.json), siem_alerts.json (générique type Wazuh/Elastic). Le README explique comment basculer vers de vraies intégrations plus tard sans changer le reste du pipeline.
